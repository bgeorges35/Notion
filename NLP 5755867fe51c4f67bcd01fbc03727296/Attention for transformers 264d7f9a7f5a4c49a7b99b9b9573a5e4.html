Computing attention

Dot product attention

DDD﻿ → Embedding size LkL_kLk​﻿ → Input sequence size LQL_QLQ​﻿ → Output sequence size

Matrix

  • KKK﻿ (keys) → (LK,D)(L_K, D)(LK​,D)﻿ embedding of input words
  • VVV﻿ (values) → (LK,D)(L_K, D)(LK​,D)﻿ here often the same as keys
  • QQQ﻿ (queries) → (LQ,D)(L_Q, D)(LQ​,D)﻿ embedding of output words

Steps

  1. We take the dot product of KKK﻿ and QQQ﻿ to get the similarity between them
  1. Pass this in softmax to have probability ⇒ it's called the attention weight matrix: WAW_AWA​﻿ (LQ,LK)(L_Q, L_K)(LQ​,LK​)﻿
  1. Weighted sum between VVV﻿ and attention weight (step 3) with a matrix multiplication

    → We weight each value viv_ivi​﻿ by it's probability that the key kik_iki​﻿ match the query

Multi-headed attention

Multi-headed attention compute multiple attention (head) that focus on different things with the same Quey / Key / Value group.

To achieve that, before computing the attention (with the scaled dot product attention), we pass each Quey / Key / Value in a dense layer (one by head).

That's the part that's gonna learn on what to focus the attention of each head (by transforming Q/K/V for this head)

The formula for the dot product attention change:

  • dkd_kdk​﻿ → Query and key dimension (for words, embedding size)

Steps

1 - Input:

For each batch, we have 3 matrices: QQQ﻿, KKK﻿ and VVV﻿ each of size (input length, embedding size)

⇒ [batch, length, d_model]

2 - Linear layer / Transpose:

For each batch, for each attention head, we pass our 3 matrices in 3 linear layer. For each batch, the result is of size: (number of head, input length, head dimension) ⇒ [batch, n_heads, length, d_head]

3 - Apply attention treating:

For each of our batch, in each of our head, we apply attention with QQQ﻿, KKK﻿ and VVV﻿ (each of dimension [input length, head dimension])

4 - Result shape:

For each of our batch, in each of our head, it give us one matrix ZZZ﻿ of dimension (input length, head dimension) ⇒ [batch, n_heads, length, d_head]

5 - Transpose / Linear layer: (or Concatenation)

For each batch, we multiply all ZZZ﻿ of each head by the "output weight matrix" W0W^0W0﻿ (in the "linear" step ??). For each batch, we have an output matrix of size (input length, embedding size) ⇒ [batch, lenght, d_model]

Types of attention

  • Encoder/decoder attention → One sentence (decoder) looks at an other one (encoder). It's what we used in 👀Attention for RNN (Neural Machine Translation)
  • Causal (self) attention → In one sentence, look at previous word from the sentence (used for generation)
  • Bi-directional self attention → In one sentence, look at previous and future word

Mask

After the dot product of QQQ﻿ (query) and KKK﻿ (keys) (step 1), we add a new mask matrix MMM﻿. A part of this matrix MMM﻿ is set to −∞-\infty−∞﻿ and the rest to 000﻿ :

  • Element from QKTQK^TQKT﻿ added with −∞-\infty−∞﻿ will be then set to a probability near 0 by the softmax function
  • Element from QKTQK^TQKT﻿ added with 000﻿ will be unchanged

⇒ It allow to "remove" (set prob to 0) some parts

Reminder, in QKTQK^TQKT﻿ Line → Query elements Column → Keys elements Cell (i,ji,ji,j﻿)→ Similarity between keys jjj﻿ and query iii﻿ or importance of a keys jjj﻿ with the query iii﻿

So here, our mask remove element where j>ij > ij>i﻿

exemple: If we want to predict the third word, our query is the second word (second line, index 1). In the keys, we only want to look at the first and second word (index 0 and 1). So in the second line, we set every word to −∞-\infty−∞﻿, except for the first and second one
